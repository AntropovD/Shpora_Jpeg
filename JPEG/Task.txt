
Нужно реализовать упрощённый вариант формата JPEG для сжатия и распаковки изображений. В качестве отправной точки вам дан пример программы, которая в однопоточном режиме сжимает изображение с использованием DCT (Discrete Cosine Transform) c потерей цвета.
Необходимо:
Распараллелить сжатие изображения по блокам

Подключить кодирование алгоритмом Хаффмана (однопоточная реализация есть в примере)

Распараллелить кодирование и декодирование

Задача минимум – выполнять все преобразования с потерей цветности изображения; максимум – сохранять информацию о цвете.
Ваша программа должна уметь принимать следующие аргументы командной строки:
-e <fileName> сжимать файл. По умолчанию данные принимаются с STDIN. Формат файла сжатого изображения вы определяете сами.

-t <N> максимальное кол-во потоков обработки (степень параллелизма). Значение по умолчанию – 1.

-w <N> размер окна DCT-преобразования. Значение по умолчанию – 8.

-q <N> процент неигнорируемых старших частот. Значение по умолчанию – 100 (никакие частоты не игнорируются).

-d <filePath> разжимать файл. По умолчанию данные принимаются с STDIN. Формат распакованного файла – 24-битный bmp.

Результат программа должна выводить на STDOUT
